Regras de entrada neste arquivo:

Todas as novas entradas devem ser datadas e podem ou não incluir nomes de código para cada versão. Cada entrada deve começar com o caractere "+", seguido da data, entre "[]" e ":". Após, as alterações devem ser descritas da forma mais clara possível.

Alterações realizadas no kernel Hexagon®:
=========================================

+ [11/01/2021]: Anterior a data de 11/01/2021, as alterações não estavam sendo documentadas desde fevereiro de 2020.
+ [22/01/2021]: Foi alterada a função paraDispositivo() em "Dev/dev.asm", visto que a passagem de um código de dispositivo em DX poderia sobrescrever o próprio código de classe de dispositivo, em DL. Para dispositivos com código de 8-bytes, o código era fornecido em AH, enquando que códigos como uma WORD eram fornecidos em DX. Agora, para padronizar, dispositivos com código de dispositivo em formato WORD serão fornecidos ao Hexagon® atraves de AX. Correções ortográficas em comentários no mesmo arquivo foram executadas. Aparentemente nenhum problema foi verificado após a mudança, uma vez que a função era de uso exclusivo dentro do próprio arquivo, para resolver nomes de dispositivo através de endereços. A data de geração do Hexagon® também poderá ser integrada à string retornada após chamada de sistema de obtenção de versão.
+ [27/01/2021]: Arquivo "Hexagon\Dev\Video\video.asm" renomeado para "Hexagon\Dev\Console\console.asm". A pasta também teve seu nome alterado. O arquivo "Hexagon\API\graficos.asm" foi movido para "Hexagon\Lib\graficos.asm".
+ [30/01/2021]: Alteração no nome de variáveis e constantes do kernel. Troca de rótulo fimKernel para Hexagon.FimCodigo, troca de BUFFER_DE_DISCO para Hexagon.CacheDisco, troca de ARGUMENTOS_PROGRAMA para Hexagon.ArgumentosProcesso e troca de Bloco_modo_VBE para Hexagon.BlocoModoVBE, com substituição de referência em todos os arquivos afetados. Alteração no sistema de fontes, com a adição de um campo de assinatura para confirmar que o arquivo fornecido é de fato uma fonte compatível. A verificação de fonte ficará a cargo de um utilitário, não será feita no Hexagon®. Alteração de versão do Kernel, de 8.45+ para 8.46A.
+ [05/03/2021]: Alterações nos nomes de rotinas do Hexagon®, substituindo nomes genéricos por nomes que localizem a função dentro da árvore do kernel. Uma alteração realizada, como exemplo, é da função usarBufferVideo1 para Hexagon.Kernel.Lib.Graficos.usarBufferVideo1, dando à rotina uma espécie de "namespace" e indicando também a origem do arquivo que contém o código, sendo, neste caso, "Hexagon/Lib/graficos.asm", com rotina localizada no interior da imagem do Hexagon®, como componente direto do kernel, presente no diretório Lib e no arquivo "graficos.asm", rotina usarBufferVideo1. Essas alterações serão realizadas aos poucos, uma vez que cada alteração em um arquivo envolve alterações em outros arquivos.
+ [11/03/2021]: As alterações dos nomes de rotina foram finalizados. Agora o código do driver de disco também emite mensagens de debug via porta serial em caso de erros relacionados ao disco. As mensagens serão expandidas para mais subsistemas do Hexagon®, bem como utilitários do Sistema.
+ [12/03/2021]: Alteração em Hexagon.Kernel.Arch.x86.Procx86.Procx86.configurarProcessador para adição das instruções finit e fwait, para inicializar a unidade de ponto flutuante.
